program "files" (module scripts), must return a buffer.fromstring of the source on require:

example:

    return buffer.fromstring[[
      int main() {
        int a = 1 + 5 + 6;
        return a;
      }
    ]]

libraries must be object files (made with the assembler), passed into the third argument of compile

so in final:
  first argument is the source,
  second argument is a root instance, so a folder, for the preprocessor to be able to parse through includes properly
  third argument is flags,
  fourth argument is a table consistent of references to modules of a library, or a buffer of the library

all flags:

    -B NUM : sets the base when passed to linker to NUM, example: -B 0x400 will tell the linker to place the final binary at 0x400, default is 0x400
    -lib : sets this to be a library file, with omits the entry label and the mandatory call to int main() from entry
    -obj : sets this to ignore linking stage (in turn ignoring -B) and return an object file
    -nasm : does not compile to object file or link, just returns the buffer containing the raw source assembly generated by codegen, ignoring -lib, -obj, and -B
